# TONY_FEEDBACK

## Purpose

This document is the living journal of Tony's evolution.

It captures real-world observations, ideas, frustrations, successes, and opportunities discovered while using Tony every day.

This is not a bug tracker.

This is not a feature request list.

This document records what it actually feels like to live and work with Tony.

Every improvement should make Tony more trustworthy, more useful, more human, and a better Chief of Staff.

---

# How to Use

Whenever something stands out while using Tony, record it.

Examples:

- Tony gave an amazing answer.
- Tony missed important context.
- Tony should have remembered something.
- Tony asked a great follow-up question.
- Tony felt robotic.
- Tony saved me time.
- Tony challenged my thinking.
- Tony made me open another app when he could have answered directly.

Small improvements become great products.
