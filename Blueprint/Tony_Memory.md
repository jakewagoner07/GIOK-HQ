# Tony Memory — Evolution Through Version 3

Tony Memory is what makes Tony a chief of staff instead of a chatbot: **durable, structured
knowledge of Jake's world.** It is the practical expression of *"your brain is for thinking, not
remembering"* — Tony remembers so Jake doesn't have to.

> **This is NOT conversation memory.** It is not a transcript log. It is structured memory —
> organized facts Tony can reason over, surface, and act on.

## The permanent contract
- **Structured, not chat.** Memory is organized into categories with typed entries, not a
  rolling conversation history.
- **A lens, never a rival copy.** Tony Memory reads from GIOK's single source of truth (registry,
  captures, workspaces). It links to the truth; it does not duplicate it (Core Principle #1, #4).
- **Transparent and correctable.** Jake can see, edit, and prune what Tony remembers. Wrong
  memories get fixed, not defended.
- **Earned, not assumed.** Memory grows from real signals (captures, completed work, corrections),
  not from guessing.

## The categories (the framework)
People · Ideas · Preferences · Business · Family · Goals · Relationships · Lessons · Patterns.

Each holds durable knowledge:
- **People** — clients, family, prospects, mentors, and what matters about each.
- **Ideas** — worth keeping and developing.
- **Preferences** — how Jake likes things done.
- **Business** — durable facts about the agency and how it runs.
- **Family** — the people closest to Jake and what matters to them.
- **Goals** — long-horizon intentions Tony keeps alive.
- **Relationships** — connection history and last-contact for key people.
- **Lessons** — what worked and what didn't.
- **Patterns** — recurring things Tony notices over time.

## Version 1 — Framework *(built in Sprint Juliet)*
- **Storage & structure only.** `tony_memory.json` exists with all nine categories, empty.
- **A Tony Memory workspace** lists the categories, their (zero) counts, and what each is for.
- No population yet — the framework is in place so later versions have a home to grow into.

## Version 2 — Population
Memory starts filling from real activity:
- **From captures:** processing a capture can add to memory — a person mentioned, an idea, a
  preference, a lesson. Tony proposes; Jake confirms.
- **From work:** completed Checkups, closed clients, and kept commitments become durable facts
  (People, Business, Relationships).
- **Surfacing:** Tony uses memory in recommendations and the Morning Brief — "Sarah's Checkup is
  due and she just had a baby" pulls from People + Relationships.
- **Editing:** a real memory management UI — review, correct, merge, prune.

## Version 3 — Reasoning & Coaching
Memory becomes the substrate Tony *thinks* with:
- **Relationship intelligence:** last-contact tracking nudges Jake before someone goes cold
  (People Matter More Than Money, enforced by memory).
- **Pattern recognition:** Tony notices recurring patterns (which outreach converts, when
  consistency slips) and coaches on them.
- **Preference-aware execution:** as Tony earns autonomy, he acts in line with remembered
  preferences (how Jake writes, when he meets, what he never does).
- **Lessons applied:** past lessons inform future recommendations, so the operation compounds.
- **Cross-workspace memory:** connects business, family, health, and goals into one coherent
  picture of Jake's life.

## Guardrails that never change
- Never fabricate a memory; unknown stays unknown.
- Never expose or act on a memory outside granted, revocable permissions.
- Never let memory become a second source of truth — it references, it doesn't own.
- People first: memory serves relationships and wellbeing before it serves metrics.

Tony Memory is the difference between an assistant that answers and a chief of staff that *knows*.
Built patiently and honestly, it's what lets Tony earn real trust over years.
