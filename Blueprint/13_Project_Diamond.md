# Project Diamond

## The Standard For Building GIOK

*This is not a feature. It is not application code. It is the product philosophy that every future
sprint, developer, AI agent, and contributor must follow. It is a permanent part of the Blueprint.*

---

## Purpose

Project Diamond marks the moment GIOK **stopped chasing features and began perfecting
experiences.**

Our goal is no longer to build the biggest software. Our goal is to build software people
**genuinely love using every day.**

## The Philosophy

Features are easy. Experiences are difficult.

Anyone can add another button. Very few products make people smile.

**Project Diamond exists to ensure quality always comes before quantity.**

## Mission

Every interaction should:

- **Save time**
- **Reduce stress**
- **Increase clarity**
- **Help someone intentionally build the life they want**

---

## The Diamond Rules

**1. Every click has a purpose.** Nothing exists simply to exist.

**2. Tony always adds value.** Tony never interrupts unless he improves the user's experience.

**3. One source of truth.** Never duplicate data.

**4. Capture first. Organize later.**

**5. The user owns everything. Always.** All personal data remains portable and exportable.

**6. Every workspace is self-contained.**

**7. Every screen answers three questions:**
- Where am I?
- What's most important?
- What's my best next action?

**8. Every widget has an action.** No dead ends.

**9. Design for ten years.** Never choose shortcuts that damage the future.

**10. People Matter More Than Money.** Every recommendation respects the user's priorities.

**11. Technology should never replace living.** It should create more time to live.

> These rules don't replace the [Core Principles](02_Core_Principles.md) — they raise the bar on
> them. Rules 3, 4, 6, and 10 are the Core Principles held to the Diamond standard; the rest define
> what "good" means for every screen and interaction.

---

## The GIOK Standard

Before any feature ships, ask:

- Is it **useful**?
- Is it **beautiful**?
- Is it **simple**?
- Is it **fast**?
- Does **Tony improve** it?
- Does it **save time**?
- Would **Jake use it every day**?
- Would **someone smile** while using this?

**If any answer is "No," the feature is not finished.**

---

## The Founder Promise

GIOK exists to help people intentionally build the life they want.

- We believe technology should **reduce complexity, not create it.**
- **AI should explain. Never control.**
- **People matter more than money.**
- **The user owns their data.**
- Every recommendation should help create **more time for what matters most.**

---

## The Ten-Year Rule

Before building anything, ask:

> **Will this still make sense ten years from now?**

If not — rethink it.

---

## Current Project Diamond Objective

- No unnecessary features.
- No unnecessary complexity.
- **Polish. Refine. Remove friction.**
- Build software people cannot imagine living without.

---

## Definition of Done

A feature is complete **only** when:

- It **works.**
- It is **intuitive.**
- It **feels polished.**
- It **follows the Blueprint.**
- It **follows the Diamond Rules.**
- It **saves time.**
- It **improves the user's experience.**

Anything less is unfinished — no matter how much code it took.
